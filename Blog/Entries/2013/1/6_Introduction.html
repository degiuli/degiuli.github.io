windows debugging series

 

Introduction

6 de janeiro de 2013

 

For now and next few posts, I will put together some information related to Windows Debugging I have been studying. The idea is to share some details to help Windows app developers and support team to find bugs in app faster. Not only when the app was shipped but also during development cycle.

The topics to be discussed you can see below in the list of posts that will be updated every time time new one is published. So, you will be able to find all series references into one single place.


Remember: this series is an introduction and you can find much more detailed information along the web. During the development of this series, I will show some interesting sites for you to improve you knowledge.


Thank you!


List of posts:

  1. *Glossary - gives brief definition of keywords and expressions used along this series.

  2. *What is Debug? - quick introduction of what I think debug is.

  3. *Debugging Tools For Windows - introduces a set of tools useful for debugging provided by the Microsoft.

  4. *Leak Diagnosis Tools - introduces two tools UMDH, part of the Debugging Tools For Windows and LeakDiag.

  5. *Application Verifier - introduces a powerful application for debugging.

  6. *Global Flags (GFlags) - introduces the gflags application used to setup application instrumentation.

  7. *ADPlus - introduces a tool to generate dump files from crash and hangs for postmortem debugging.

  8. *WinDbg Part I - introduces the WinDbg tool and one of its most important command !analyze.

  9. *Memory Corruption - describes what is a memory corruption and how to detect it.

  10. *Stack Corruption - introduces what is stack data structure and its common issues.

  11. *Heap Corruption - introduces what is heap and its common issues.

  12. *WinDbg Part II - this introduces more useful commands supported by the WinDbg tool that helps to identify an issue in dump file.
